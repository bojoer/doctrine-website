_hydrateRow

protected void _hydrateRow(array data, array cache, array result)

Hydrates a single row in an SQL result set.

Internal:
First, the data of the row is split into chunks where each chunk contains data that belongs to a particular component/class. Afterwards, all these chunks are processed, one after the other. For each chunk of class data only one of the following code paths is executed: Path A: The data chunk belongs to a joined/associated object and the association is collection-valued. Path B: The data chunk belongs to a joined/associated object and the association is single-valued. Path C: The data chunk belongs to a root result element/object that appears in the topmost level of the hydrated result. A typical example are the objects of the type specified by the FROM clause in a DQL query.
Parameters:
data - The data of the row to process.
